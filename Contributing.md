# Contributing to Sentiment Analysis of Product Reviews

Thank you for your interest in contributing to this project! Please follow the guidelines below to keep the repository clean and consistent.

---

## Getting Started

1. **Fork** the repository
2. **Clone** your fork locally:
   ```bash
   git clone https://github.com/your-username/Sentiment_Analysis_of_Product_Reviews.git
   ```
3. Create a new branch for your feature or fix:
   ```bash
   git checkout -b feature/your-feature-name
   ```

---

## Project Structure

Please place files in the correct folders:

| Folder | Purpose |
| --- | --- |
| `code/` | Jupyter notebooks and Python scripts |
| `models/` | Saved model files (.pkl, .pt, .h5) |
| `Data/` | Dataset references or sample data |
| `reports/classification_report/` | Per-model classification reports |
| `results/plots/` | Output charts and visualizations |

---

## Code Guidelines

- Use clear, descriptive variable and function names
- Add comments explaining key steps in notebooks
- Keep notebooks clean — clear all outputs before committing
- Follow PEP 8 style for Python scripts

---

## Commit Guidelines

Write clear commit messages:

```
feat: add DistilBERT fine-tuning notebook
fix: correct label encoding in preprocessing
docs: update README with model results
```

---

## Pull Request Process

1. Ensure your branch is up to date with `main`
2. Describe what your PR does and why
3. Link any related issues
4. Wait for review from a team member before merging

---

## Reporting Issues

Open an issue on GitHub with:
- A clear title and description
- Steps to reproduce (if a bug)
- Expected vs actual behaviour
